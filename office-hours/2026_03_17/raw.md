Nnamdi — 11:47 AM
The paper introduces OpenClaw-RL, a framework for training an agent from its normal live interactions, rather than only from a fixed offline dataset. The central idea is that after every agent action, the next state already contains learning signal: a user reply, a tool result, a graphical user interface change, a terminal output, or a test verdict. The authors split this signal into two parts.
 First, evaluative signal, which says whether the action was good or bad, and is converted into scalar rewards using a process reward model judge. 
Second, directive signal, which says how the action should have been different, and is turned into token-level supervision through their method called Hindsight-Guided On-Policy Distillation. 
The system is asynchronous: serving, rollout collection, judging, and training run in parallel, so the model can keep handling live requests while being improved. They test this both for personal conversational agents and for more general agents operating in terminal, graphical user interface, software engineering, and tool-call settings. 
Their main claim is that combining scalar reinforcement learning with token-level corrective supervision gives better improvement than using either alone, especially for long-horizon agent tasks and personalized assistants.
